### 🚀 Git Worktree 기반 작업 가이드라인

#### 1. 피처(Feature) 브랜치 작업

- **목적:** 새로운 기능 추가 및 단위 개발.
- **워크플로우:**

1. **워크트리 생성:** 메인(또는 develop) 브랜치에서 새로운 피처 브랜치를 위한 워크트리를 생성합니다.

```bash
git worktree add ../project-feature-[기능명] -b feature/[기능명]

```

2. **독립 개발:** 생성된 디렉토리로 이동하여 클로드 코드(또는 IDE)를 실행하고 개발을 진행합니다.
3. **커밋 및 푸시:** 작업이 완료되면 해당 워크트리 내에서 커밋 후 원격 저장소에 푸시합니다.

```bash
git add .
git commit -m "Feat: [기능명] 구현 완료"
git push origin feature/[기능명]

```

#### 2. 메인(Main/Develop) 브랜치 관리

- **목적:** 통합 빌드 에러 확인, 버그 수정, PR(Pull Request) 검토 및 병합.
- **워크플로우:**

1. **병합(Merge):** 피처 브랜치 작업이 끝나면 메인 워크트리에서 피처 브랜치를 병합합니다. (또는 GitHub/GitLab에서 PR 승인)
2. **검증:** 메인 워크트리는 항상 '배포 가능한 상태'를 유지해야 합니다. 병합 후 빌드 에러나 예기치 못한 버그가 있는지 확인합니다.
3. **긴급 수정:** 메인에서 발견된 버그는 메인 워크트리에서 직접 고치거나, 별도의 `hotfix` 워크트리를 만들어 빠르게 대응합니다.

#### 3. 작업 완료 후 정리

- **목적:** 불필요한 디렉토리 및 워크트리 참조 제거.
- **방법:** 피처 브랜치가 메인에 병합 완료되면 워크트리를 삭제합니다.

```bash
git worktree remove project-feature-[기능명]

```
